# Renewlet v0.1.7

Renewlet v0.1.7 continues improving background scheduling performance and makes copy actions more reliable. Notification and auto-renewal behavior stays the same for users, but background jobs now avoid unnecessary reads when related features are disabled or there is nothing to process. Calendar feed, public page, and error-detail copy actions now handle browser Clipboard API capability differences consistently instead of exposing raw browser errors.

[中文 ->](./v0.1.7-zh.md)

---

## Read Before Upgrading

Docker users do not need any manual migration when upgrading from v0.1.6. Cloudflare users should keep using the normal deployment flow; this release includes a new scheduler state table and indexes.

---

## Improvements

- Background notifications and auto-renewal now track whether each user has subscriptions that need scheduled checks, so disabled auto-renewal or repeat reminders no longer cause unrelated candidate reads.
- Adding, editing, deleting, or importing subscriptions refreshes scheduler state automatically, keeping later background checks aligned with current subscription settings.
- Copying calendar feed URLs, public status page URLs, and error details now gives a clear fallback message when the browser Clipboard API is unavailable or denied, and Renewlet tries to select the visible URL or text for manual copying.

---

## Bug Fixes

- Fixed calendar feed URL copying failing in some browsers because of Clipboard API compatibility differences and showing an English `writeText` error ([#29](https://github.com/zhiyingzzhou/renewlet/issues/29), thanks [@hkl8080](https://github.com/hkl8080)).

---

## Reliability and Performance

- On Cloudflare, notification and auto-renewal cron jobs now check whether candidate subscriptions need to be read first, reducing D1 rows read.
- Docker/PocketBase installs also avoid background idle work that previously added unnecessary SQLite/PocketBase I/O and CPU usage.
