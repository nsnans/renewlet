# Renewlet v0.1.0

Renewlet v0.1.0 focuses on calendar feeds, data migration, Cloudflare deployment, in-app updates, and a set of fixes for upgrade-time rough edges.

[中文 ->](./v0.1.0-zh.md)

---

## Read Before Upgrading

If you deployed Renewlet before 2026-06-04, open the old version before upgrading and export `subscriptions.json` from Export subscriptions. After upgrading, import it from Import data.

`subscriptions.json` is only for subscription migration. It is not a full instance backup. Keep a backup of `.env`, `docker-compose.yml`, and `data/` before upgrading.

If you are still on the old Docker layout, run `docker compose pull && docker compose up -d` once to move to the new layout. Later stable releases can use the version badge at the top of Renewlet to open System Update.

---

## New Features

- Renewal calendar: add global private ICS feeds and per-subscription feeds for calendar apps ([#9](https://github.com/zhiyingzzhou/renewlet/issues/9), thanks [@Chuyiyue](https://github.com/Chuyiyue)).
- Data migration: support new Renewlet backup import/export, old `subscriptions.json` imports, and migration from Wallos JSON, backup.zip, and SQLite data.
- Cloudflare Workers deployment: add a Cloudflare runtime path that does not depend on Go/PocketBase.
- System Update: Docker release images can update from inside Renewlet; Cloudflare and source deployments show the matching manual upgrade path.

---

## Improvements

- Mobile subscription lists, filters, settings, and notification dialogs are easier to use, with steadier loading states.
- Logo selection now brings upload, image URLs, built-in icons, and favicon suggestions into one flow.
- Notification history, subscription lists, and exchange-rate fallback handling were tightened up so everyday use is less likely to get stuck on empty states or remote failures.

---

## Bug Fixes

- Fix settings navigation highlighting when scrolling through sections.
- Fix notification history showing raw `email` instead of the localized channel name.
- Fix Cloudflare showing success for restarts it cannot actually perform.
- Fix ICS folded-line handling and fallback labels for built-in categories and payment methods in public calendar feeds.

---

## Security and Reliability

- Add Go and Cloudflare `/api/app/ready` endpoints to check whether the runtime is actually usable.
- Harden outbound notification URLs by rejecting unsafe protocols, userinfo, localhost, private networks, and IP literals.
- Clear stale sessions after auth failures, and route frontend exceptions through the Error Boundary and error reporting.
