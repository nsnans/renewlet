# Renewlet v0.1.3

Renewlet v0.1.3 补上了演示模式、上传图标管理和订阅分类多选筛选，也修正了一些续费提醒、通知历史和设置页显示问题。

[English ->](./v0.1.3-en.md)

---

## 升级前请先读

从 v0.1.2 升级不需要手动迁移。Docker release 镜像可以继续从页面顶部版本号进入“系统更新”；Cloudflare 部署仍建议先同步 fork，再确保 D1 migrations 先于 Worker 发布完成。

Docker Demo Mode 需要在 `.env` 中显式设置 `RENEWLET_DEMO_MODE=true`。未开启时，普通实例不会进入演示数据、演示账号或外部发送禁用流程。

---

## 新增功能

- 演示模式：可以用同一套 Docker 镜像部署公开演示实例。访客使用演示账号登录，演示数据会自动重置，通知发送、云备份等真实外部操作不会执行。
- 上传图标管理：设置页现在可以查看和管理已上传的图标，未被订阅或支付方式使用的图标可以删除。
- 分类多选筛选：订阅列表支持同时选择多个分类。桌面端可以连续勾选，移动端在抽屉中确认后应用。

---

## 优化改进

- 首页和侧边栏的“即将续费/到期”现在按每个订阅自己的提醒天数显示。
- 订阅卡片只在仍需关注续费或到期时显示剩余天数；暂停、取消和买断订阅不再显示容易误解的倒计时。
- 设置页恢复了分类管理、支付方式管理和上传图标管理入口。
- 时间选择器修正了横向偏移和滚轮停靠问题。
- AI 识别、云备份和通知测试等错误详情更集中，排查时能看到必要的脱敏响应内容。

---

## Bug 修复

- 通知历史只展示用户需要查看的发送结果，不再夹带排程信息。
- 正在被订阅 Logo 或支付方式使用的上传图标不会被误删。
- 首页统计和即将续费列表对提醒窗口的判断保持一致。
- Cloudflare 自管部署在缺少必要 secrets 时只做检查和构建，不会误改远端资源。
- 公开链接在代理、隧道或自定义域名环境下生成地址更准确。

---

## 隐私与可靠性

- 演示模式会隔离真实外部副作用，适合公开试用实例。
- 通知、AI 服务和云备份的上游错误会隐藏密钥、密码和签名等敏感内容。
- 上传图标删除前会检查使用情况，避免订阅或支付方式突然丢失图标。
