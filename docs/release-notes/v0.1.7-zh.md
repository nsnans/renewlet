# Renewlet v0.1.7

Renewlet v0.1.7 继续改善后台调度性能，并让复制链接的体验更稳。通知和自动续订的用户语义保持不变，但关闭相关功能或没有待处理订阅时，后台任务会少做无效读取；日历订阅、公开页面和错误详情里的复制入口也统一处理不同浏览器的剪贴板 API 能力差异，不再暴露生硬的浏览器错误。

## 升级前请先读

Docker 用户从 v0.1.6 升级不需要手动迁移。Cloudflare 用户请继续按现有部署流程发布；本版本包含新的调度状态表和索引。

---

## 优化改进

- 后台通知和自动续订会记录每个用户是否存在需要检查的订阅，关闭自动续订或重复提醒后，不再反复读取无关候选订阅。
- 新增、修改、删除或导入订阅后，后台调度状态会自动刷新，避免后续检查沿用旧的订阅状态。
- 复制日历订阅地址、公开状态页地址和错误详情时，浏览器剪贴板 API 不可用或被拒绝会给出清晰提示，并尽量选中页面上已有的地址或文本，方便手动复制。

---

## Bug 修复

- 修复日历订阅地址复制在部分浏览器中因剪贴板 API 兼容性差异失败，并显示英文 `writeText` 错误的问题（[#29](https://github.com/zhiyingzzhou/renewlet/issues/29)，感谢 [@hkl8080](https://github.com/hkl8080)）。

---

## 可靠性与性能

- Cloudflare 部署下，通知和自动续订 cron 会先判断是否有必要读取候选订阅，减少 D1 rows read。
- Docker/PocketBase 部署同样减少后台空跑带来的 SQLite/PocketBase I/O 和 CPU 消耗。
