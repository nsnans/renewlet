# Renewlet v0.1.3

Renewlet v0.1.3 adds Demo Mode, uploaded-icon management, and multi-category subscription filters, and fixes several renewal reminder, notification history, and settings display issues.

[中文 ->](./v0.1.3-zh.md)

---

## Read Before Upgrading

No manual migration is required when upgrading from v0.1.2. Docker release images can continue to update from the version badge at the top of Renewlet. For Cloudflare deployments, sync your fork and make sure D1 migrations run before the Worker is published.

Docker Demo Mode must be enabled explicitly in `.env` with `RENEWLET_DEMO_MODE=true`. Normal instances do not use demo data, the demo account, or disabled external sends unless that setting is enabled.

---

## New Features

- Demo Mode: deploy a public demo instance with the same Docker image. Visitors sign in with the demo account, demo data resets automatically, and real external actions such as notification sends and cloud backups are disabled.
- Uploaded-icon management: Settings can now show uploaded icons and delete icons that are not used by subscriptions or payment methods.
- Multi-category filters: the subscription list can filter by several categories at once. Desktop users can keep selecting in the popover, and mobile users apply the selection from the drawer.

---

## Improvements

- Dashboard and sidebar upcoming-renewal views now use each subscription's own reminder days.
- Subscription cards show remaining days only when the subscription still needs renewal or expiry attention; paused, cancelled, and lifetime records no longer show misleading countdowns.
- Settings restores the category management, payment-method management, and uploaded-icon management entry points.
- TimePicker fixes horizontal drift and wheel snapping.
- AI recognition, cloud backup, and notification test error details now show the needed redacted response details in one place.

---

## Bug Fixes

- Notification history no longer shows internal scheduling fields, and keeps the send result focused on what users need to review.
- Uploaded icons cannot be deleted while a subscription Logo or payment method still uses them.
- Dashboard stats and upcoming-renewal lists now use the same reminder-window rules.
- Self-managed Cloudflare deployments only check and build when required secrets are missing, instead of changing remote resources.
- Public links are generated more accurately behind proxies, tunnels, and custom domains.

---

## Privacy and Reliability

- Demo Mode isolates real external side effects, so public demo instances are safer to share.
- Upstream errors from notifications, AI providers, and cloud backups redact secrets, passwords, signatures, and similar sensitive values.
- Uploaded-icon deletion checks usage first, so subscriptions and payment methods do not unexpectedly lose icons.
