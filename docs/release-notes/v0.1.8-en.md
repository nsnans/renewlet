# Renewlet v0.1.8

Renewlet v0.1.8 adds family cost sharing ([zhiyingzzhou/renewlet#32](https://github.com/zhiyingzzhou/renewlet/pull/32), thanks [@Shy0o](https://github.com/Shy0o)). Add shared members to a subscription, choose how the cost is split, and Renewlet will show your share, the family contribution, and the recoverable amount. This release also improves the Cloudflare one-click upgrade path, currency selection, and long subscription filters.

---

## Read Before Upgrading

Docker users do not need any manual migration when upgrading from v0.1.7.

Cloudflare users should use the existing deploy flow; this release includes the data migration required for family cost sharing.

Cloudflare one-click deploy users should upgrade from the generated repository by running `Sync Renewlet Upstream`. Do not click the one-click deploy button again. If an older generated repository does not have this entry yet, add it once from the [Cloudflare deploy documentation](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.md#existing-one-click-deploy-users).

---

## New Features

- Family cost sharing: add shared members to a subscription, then choose an equal split or custom amounts.
- Members can use different currencies. Subscription cards and details show your share, the family contribution, and the recoverable amount.
- Cost-sharing data is saved with the subscription and stays included during import and export.

---

## Improvements

- Cloudflare one-click deploy now has a follow-up upgrade entry from the generated repository.
- Currency names and codes are clearer, and common currencies are easier to find.
- Currency pickers follow the order from settings. Currencies already used by older subscriptions but disabled in settings are shown for context, not offered as new choices.
- When there are many categories or tags, subscription filter popovers scroll inside the list area, keeping the footer actions visible.
