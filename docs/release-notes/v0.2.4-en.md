# Renewlet v0.2.4

Renewlet v0.2.4 improves outbound proxy support, startup reliability, and supported import formats for self-hosted Docker deployments.

[中文 ->](./v0.2.4-zh.md)

---

## Read Before Upgrading

Cloudflare one-click deploy users should still upgrade from the generated repository by running `Sync Renewlet Upstream`. Do not click the one-click deploy button again. If an older generated repository does not have this entry yet, add it once from the [Cloudflare deploy documentation](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.md#existing-one-click-deploy-users).

---

## New Features

- Docker deployments now support `HTTP_PROXY`, `HTTPS_PROXY`, `NO_PROXY`, and their lowercase variants for server-side requests to Telegram, AI providers, GitHub Release checks, built-in icon indexes, WebDAV, and S3-compatible storage.

---

## Improvements

- Docker and local startup now prepare frontend static files first, making direct visits to pages like `/status` and Settings more reliable.
- Imports support current Renewlet export files and Wallos files; very early Renewlet export formats are no longer accepted as import sources.

---

## Bug Fixes

- Fix a startup crash caused by route conflicts on some Go/PocketBase versions.
- Fix page errors such as `/status` when frontend static files were missing.
