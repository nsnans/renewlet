# Renewlet v0.1.1

Renewlet v0.1.1 adds AI subscription recognition, public subscription status pages, manual and automatic renewal handling, richer billing-cycle models, ServerChan notifications, and a set of import, mobile, and runtime refinements.

[中文 ->](./v0.1.1-zh.md)

---

## Read Before Upgrading

No manual migration is required when upgrading from v0.1.0. Docker release images can continue to update from the version badge at the top of Renewlet. For Cloudflare deployments, sync your fork and make sure D1 migrations run before the Worker is published.

AI recognition sends recognition content only when you configure and actively use a third-party AI provider. If you do not use AI recognition, no AI configuration is required.

---

## New Features

- AI subscription recognition: paste notes, CSV/TSV data, table text, or upload PNG/JPG/WebP screenshots to generate editable subscription drafts before importing.
- Public subscription status pages: generate a shareable public page, control visibility per subscription, and choose whether prices are shown.
- Manual and automatic renewals: recurring subscriptions can be renewed manually or set to auto-renew so background maintenance advances their next billing date.
- Richer billing cycles: add custom cycle units plus one-time purchase records for lifetime buyouts or fixed service terms.
- ServerChan notifications: add support for ServerChan Turbo and ServerChan 3 SendKeys.

---

## Improvements

- Reminder settings are more flexible: subscriptions can disable due reminders, and one-time buyouts default to long-lived records instead of renewal reminders.
- Import and export now preserve newer subscription fields such as pinned state, disabled reminders, custom cycles, one-time terms, and auto-renewal.
- AI drafts can be checked, previewed, and imported in separate steps; uncertain price, currency, cycle, and date fields must be corrected before import.
- Logo handling, public pages, and the mobile AI recognition dialog were tightened up so adding and reviewing subscriptions is smoother on phones.

---

## Bug Fixes

- Fix inconsistent handling of one-time purchases and fixed terms across reminders, amortized statistics, calendar feeds, and imports.
- Fix Cloudflare fresh-database migrations, scheduled-notification error handling, and local development command edge cases.
- Fix fallback details for built-in categories, payment methods, Logos, and subscription statuses in public calendar and status views.
- Fix import-preview asset uploads, field validation, and error messages so issues are easier to catch before writing data.

---

## Security and Reliability

- Public status pages stay restrained: revoked links stop working, hidden subscriptions stay hidden, and hidden prices are not included in public responses.
- Private Logos do not turn the public page into a file browser; only images used by public subscriptions can be shown.
- AI recognition sends input to your configured provider only after you click recognize. Renewlet does not analyze subscription data in the background.
- Docker and Cloudflare deployments now run the same checks around AI recognition, public pages, and renewal jobs.
