# Renewlet v0.2.4

Renewlet v0.2.4 改进 Docker 自托管的出站代理、启动稳定性和导入支持范围。

[English ->](./v0.2.4-en.md)

---

## 升级前请先读

Cloudflare 一键部署用户仍然从生成仓库运行 `Sync Renewlet Upstream` 升级，不要重新点一键部署按钮。旧生成仓库如果还没有这个入口，按 [Cloudflare 部署文档](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.zh-CN.md#旧一键部署用户) 一次性补上。

---

## 新功能

- Docker 部署支持 `HTTP_PROXY`、`HTTPS_PROXY`、`NO_PROXY` 以及对应的小写变量，用于服务端访问 Telegram、AI provider、GitHub Release、内置图标索引、WebDAV 和 S3 兼容存储。

---

## 改进

- Docker 和本地启动会先准备前端静态文件，直接访问 `/status`、设置页等前端页面更稳定。
- 导入入口支持当前 Renewlet 导出文件和 Wallos 文件；很早期的 Renewlet 导出格式不再作为导入来源。

---

## Bug 修复

- 修复部分 Go/PocketBase 版本下路由冲突导致的启动崩溃。
- 修复缺少前端静态文件时 `/status` 等页面访问异常。
